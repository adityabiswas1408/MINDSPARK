# Debug Handoff Workflow

> Use this when:
> - Blocked on a bug after two genuine fix attempts
> - A phase is complete and needs Claude Code precision review
> - Switching from Antigravity (bulk generation) to Claude Code (precision debugging)

---

## Step 1 — Capture the Error Completely

Run this and copy the complete output — every line, no truncation:

```bash
npm run tsc 2>&1
```

Do not summarise the errors. Paste the full tsc output.

---

## Step 2 — Write PROGRESS.md With Full Debug Context

```markdown
# MINDSPARK — Development Progress

## Current Phase
Phase X — [Name]

## Last Completed Task
[Exact task text from docs/17_task-breakdown.md — copy verbatim]

## Files Written This Session
- src/path/to/every/file.ts
[complete list — no omissions]

## Next Task
[Exact next unchecked [ ] task — or "Phase X complete, hand off for review"]

## Current Errors
```
[PASTE COMPLETE npm run tsc OUTPUT HERE — EVERY LINE — NO TRUNCATION]
```

## Error Location
File: [exact filepath]
Line: [line number]
Error: [exact TypeScript error message]

## What I Tried
Attempt 1: [exactly what you tried — be specific about what you changed]
Attempt 2: [exactly what you tried — be specific]

## My Hypothesis
[What I think the root cause is — even if uncertain]

## What Claude Code Should Do
[Specific instruction — e.g.:
"Fix the TypeScript error at line 47 of src/app/actions/assessment-sessions.ts.
The submitAnswer action is calling student_answers.question_index but that column
does not exist — see docs/11_database.md §12 for the correct column name question_id UUID."
OR
"Review all Phase 4 Server Actions against docs/12_api-contracts.md. Check every
action signature, every error code, and every side effect. Fix any discrepancies."]

## Relevant Canonical Source
[Which doc Claude Code should read: e.g. "docs/11_database.md §12 for student_answers schema"]
```

---

## Step 3 — Commit Current State

Commit everything — even with errors. Claude Code needs to see the actual state.

```bash
git add -A
git commit -m "WIP: Phase X — [brief description] — handing off to Claude Code

Error: [paste the first tsc error line]
Details: see PROGRESS.md ## Current Errors"
git push origin main
```

---

## Step 4 — Tell the User Exactly This

```
"I am handing this off to Claude Code. PROGRESS.md is fully updated.

To continue in Claude Code:
1. Open terminal in the mindspark/ folder
2. Run: claude
3. Paste this first message:

'Read CLAUDE.md completely. Then read PROGRESS.md completely.
The error is in ## Current Errors. The canonical source is [doc].
Fix the error. Run npm run tsc. Confirm 0 errors.
Then commit: git add -A && git commit -m "fix: [describe what was fixed]"'
"
```

---

## Step 5 — Common Handoff Templates

Copy the appropriate message for the type of issue:

### TypeScript error in a new file
```
Read CLAUDE.md completely. Then read PROGRESS.md.
Fix the TypeScript error documented in ## Current Errors.
The file is [filepath].
Before fixing, read @docs/11_database.md §[relevant section] — the error is a
column/table/RPC name mismatch.
After fixing: npm run tsc → 0 errors. Then commit.
```

### Supabase RLS policy not working correctly
```
Read CLAUDE.md completely. Then read PROGRESS.md.
The RLS policy in migration [X] is not correctly restricting access.
Read @docs/11_database.md §19 for the correct policy patterns.
CRITICAL: use (auth.jwt() -> 'app_metadata') ->> 'role' — never auth.jwt() ->> 'role'.
Test after fixing: student JWT must not SELECT another student's submissions.
Then: npx supabase db push. Commit the fixed migration.
```

### Phase 8 assessment engine bug
```
Read CLAUDE.md completely.
Read @docs/09_fsd.md completely.
Read @docs/13_exam-engine-spec.md completely.
Then read PROGRESS.md.
The bug is in ## Current Errors.
Fix it. Run all self-verification commands from CLAUDE.md §SELF-VERIFICATION COMMANDS.
Every grep must return 0 results. npm run tsc → 0 errors. Then commit.
```

### Phase complete — full review handoff
```
Read CLAUDE.md completely. Read PROGRESS.md.
Phase [X] has been generated by Antigravity. Please review all Phase [X] files.

Review checklist:
1. npm run tsc → must be 0 errors
2. Check every item in CLAUDE.md §NEVER — zero violations
3. Check every item in CLAUDE.md §HARD RULES — verify with code examples
4. For database files: check against docs/11_database.md canonical column names
5. For API files: check against docs/12_api-contracts.md signatures and payloads
6. For UI files: check CLAUDE.md §DESIGN SYSTEM colour table

Fix all violations found. Run npm run tsc after each fix. Commit the clean version.
```

---

## Step 6 — After Claude Code Returns

When returning to Antigravity after Claude Code has resolved the issue:

1. Pull the latest changes: git pull origin main
2. Run /start-phase workflow from Step 1 (read CLAUDE.md fresh)
3. PROGRESS.md should have been updated by Claude Code with the fix summary
4. Verify npm run tsc → 0 errors
5. Find the next unchecked task in docs/17_task-breakdown.md and continue
