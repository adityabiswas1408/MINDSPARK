## 1. Schema Conventions

| Convention | Rule |
|-----------|------|
| Primary keys | `UUID` generated by `gen_random_uuid()` — no serial integers |
| Timestamps | All stored as `TIMESTAMPTZ` in UTC. Display converted in application layer using `date-fns-tz` |
| Soft deletes | `deleted_at TIMESTAMPTZ NULL` — never `DELETE` curriculum or student entities |
| Row-Level Security | Enabled (`ALTER TABLE … ENABLE ROW LEVEL SECURITY`) on every table |
| Naming | `snake_case` throughout — tables, columns, functions, policies |
| Migrations | All idempotent — `CREATE TABLE IF NOT EXISTS`, `CREATE POLICY IF NOT EXISTS` |
| Auth | `auth.uid()` → current user UUID · `(auth.jwt() -> 'app_metadata') ->> 'role'` → current role claim (custom app role). Note: the top-level `(auth.jwt() -> 'app_metadata') ->> 'role'` always returns `'authenticated'` (PostgREST DB role) — it does NOT return `'admin'`/`'teacher'`/`'student'`. All RLS policies that check role MUST use the `app_metadata` path or a custom JWT hook that promotes it. |

## 1.5 Table: `profiles`

> **Audit addition (Batch 1):** This table was referenced in 10+ RLS policies throughout this document but had no DDL. Added here as the dependency of all institution-scoped access policies. Must be migration 002 — all subsequent tables depend on it.

Central user profile table. Every authenticated user (admin, teacher, student) has one row. Links Supabase Auth (`auth.users`) to institution and role metadata.

### Columns

| Column | Type | Nullable | Default | Constraints |
|--------|------|----------|---------|-------------|
| `id` | UUID | NO | — | PRIMARY KEY — matches `auth.users.id` exactly |
| `institution_id` | UUID | NO | — | FK → institutions(id) ON DELETE RESTRICT |
| `role` | TEXT | NO | — | CHECK IN ('admin','teacher','student') |
| `email` | TEXT | NO | — | UNIQUE — mirrors auth.users.email |
| `full_name` | TEXT | NO | — | Display name |
| `avatar_url` | TEXT | YES | NULL | Storage object URL |
| `forced_password_reset` | BOOLEAN | NO | `FALSE` | Set TRUE on CSV import; cleared on first manual login |
| `locked_at` | TIMESTAMPTZ | YES | NULL | Non-NULL = account locked |
| `version_seq` | INTEGER | NO | `0` | Incremented on profile/level change — cache busting |
| `created_at` | TIMESTAMPTZ | NO | `NOW()` | |
| `updated_at` | TIMESTAMPTZ | NO | `NOW()` | Updated by trigger |

### DDL

```sql
CREATE TABLE IF NOT EXISTS profiles (
  id                     UUID         PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  institution_id         UUID         NOT NULL REFERENCES institutions(id) ON DELETE RESTRICT,
  role                   TEXT         NOT NULL CHECK (role IN ('admin','teacher','student')),
  email                  TEXT         NOT NULL UNIQUE,
  full_name              TEXT         NOT NULL,
  avatar_url             TEXT,
  forced_password_reset  BOOLEAN      NOT NULL DEFAULT FALSE,
  locked_at              TIMESTAMPTZ,
  version_seq            INTEGER      NOT NULL DEFAULT 0,
  created_at             TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at             TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

### Indexes

| Index | Type | Columns | Rationale |
|-------|------|---------|-----------|
| `profiles_pkey` | BTREE | `id` | Primary key (auto) |
| `profiles_institution_idx` | BTREE | `institution_id` | Institution-scope queries used by most RLS policies |
| `profiles_email_idx` | BTREE UNIQUE | `email` | Auth login lookup |

### RLS Policies

**Plain English:** Every authenticated user can read their own profile. Admins can read all profiles within their institution. Only service-role may insert or update profiles (creation managed via Supabase Auth hooks).

```sql
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "profiles_select_own" ON profiles
  FOR SELECT TO authenticated
  USING (id = auth.uid());

CREATE POLICY "profiles_admin_read_institution" ON profiles
  FOR SELECT TO authenticated
  USING (
    (auth.jwt() -> 'app_metadata') ->> 'role' = 'admin'
    AND institution_id = (SELECT institution_id FROM profiles WHERE id = auth.uid())
  );

-- INSERT / UPDATE / DELETE: service-role only
-- Profile rows are created by the auth.users on_signup trigger (service-role)
```

## 7. Table: `cohort_history`

**Temporal join table.** Records every teacher-student-cohort assignment with validity timestamps. Enables teachers to retain historical access to data for students they previously taught, even after reassignment.

### Columns

| Column | Type | Nullable | Default | Constraints |
|--------|------|----------|---------|-------------|
| `id` | UUID | NO | `gen_random_uuid()` | PRIMARY KEY |
| `teacher_id` | UUID | NO | — | FK → profiles(id) |
| `student_id` | UUID | NO | — | FK → students(id) |
| `cohort_id` | UUID | NO | — | FK → cohorts(id) |
| `valid_from` | TIMESTAMPTZ | NO | `NOW()` | Assignment start |
| `valid_to` | TIMESTAMPTZ | YES | NULL | NULL = currently active |
| `created_at` | TIMESTAMPTZ | NO | `NOW()` | |

### DDL

```sql
CREATE TABLE IF NOT EXISTS cohort_history (
  id          UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  teacher_id  UUID         NOT NULL REFERENCES profiles(id),
  student_id  UUID         NOT NULL REFERENCES students(id),
  cohort_id   UUID         NOT NULL REFERENCES cohorts(id),
  valid_from  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  valid_to    TIMESTAMPTZ,                              -- NULL = currently active
  created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

### Indexes

| Index | Type | Columns | Rationale |
|-------|------|---------|-----------|
| `cohort_history_teacher_idx` | BTREE | `(teacher_id, valid_from, valid_to)` | Temporal join queries |
| `cohort_history_student_idx` | BTREE | `student_id` | Reverse lookup |
| `cohort_history_active_idx` | PARTIAL | `teacher_id WHERE valid_to IS NULL` | Current assignments |

### Temporal Join RLS Pattern

**Plain English:** A teacher may read any submission belonging to a student they have EVER taught, provided the teaching period overlaps with when the exam session started.

```sql
-- Used in submissions, student_answers, and related tables
-- Teacher access check: temporal overlap with student's session started_at
-- NOTE: this is the example policy pattern. The authoritative implementation
-- is in §11.5 assessment_sessions RLS policies.

CREATE POLICY "submissions_teacher_temporal" ON submissions
  FOR SELECT TO authenticated
  USING (
    (auth.jwt() -> 'app_metadata') ->> 'role' = 'teacher'
    AND student_id IN (
      SELECT DISTINCT ch.student_id
      FROM cohort_history ch
      WHERE ch.teacher_id = auth.uid()
        AND ch.valid_from  <= submissions.started_at
        AND (ch.valid_to IS NULL OR ch.valid_to >= submissions.started_at)
    )
  );
```

### Trigger: auto-close previous assignment on new cohort assignment

```sql
CREATE OR REPLACE FUNCTION close_previous_cohort_assignment()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
  -- When a new record is inserted, close any open prior assignments for this student+teacher
  UPDATE cohort_history
  SET valid_to = NOW()
  WHERE student_id = NEW.student_id
    AND teacher_id = NEW.teacher_id
    AND valid_to IS NULL
    AND id <> NEW.id;
  RETURN NEW;
END;
$$;

CREATE TRIGGER trg_close_previous_cohort
  AFTER INSERT ON cohort_history
  FOR EACH ROW
  EXECUTE FUNCTION close_previous_cohort_assignment();
```

### RLS Policies

```sql
ALTER TABLE exam_papers ENABLE ROW LEVEL SECURITY;

-- Students: read LIVE papers at their level only
CREATE POLICY "exam_papers_student_live" ON exam_papers
  FOR SELECT TO authenticated
  USING (
    (auth.jwt() -> 'app_metadata') ->> 'role' = 'student'
    AND status = 'LIVE'
    AND level_id = (SELECT level_id FROM students WHERE profile_id = auth.uid())

-- Admin: full access within institution
CREATE POLICY "exam_papers_admin_all" ON exam_papers
  FOR ALL TO authenticated
  USING ((auth.jwt() -> 'app_metadata') ->> 'role' = 'admin'
    AND institution_id = (SELECT institution_id FROM profiles WHERE id = auth.uid()))
  WITH CHECK ((auth.jwt() -> 'app_metadata') ->> 'role' = 'admin');

-- Teacher: read PUBLISHED/LIVE/CLOSED papers at their cohort's level
CREATE POLICY "exam_papers_teacher_read" ON exam_papers
  FOR SELECT TO authenticated
  USING (
    (auth.jwt() -> 'app_metadata') ->> 'role' = 'teacher'
    AND level_id IN (
      SELECT c.level_id FROM cohorts c WHERE c.teacher_id = auth.uid()
    )
  );
```

### `bulk_import_students`

```sql
CREATE OR REPLACE FUNCTION bulk_import_students(
  p_institution_id  UUID,
  p_level_id        UUID,
  p_cohort_id       UUID,
  p_students        JSONB   -- array of {full_name, roll_number, date_of_birth}
)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_student       JSONB;
  v_profile_id    UUID;
  v_inserted      INTEGER := 0;
  v_skipped       INTEGER := 0;
  v_errors        JSONB   := '[]'::JSONB;
  v_i             INTEGER := 0;
BEGIN
  FOR v_student IN SELECT * FROM jsonb_array_elements(p_students)
  LOOP
    v_i := v_i + 1;
    BEGIN
      -- Create Supabase Auth user (service-role only)
      -- Profile + student row in one atomic block
      INSERT INTO students (
        institution_id, level_id, cohort_id,
        roll_number, full_name, date_of_birth
      ) VALUES (
        p_institution_id,
        p_level_id,
        p_cohort_id,
        v_student->>'roll_number',
        v_student->>'full_name',
        (v_student->>'date_of_birth')::DATE
      );
      v_inserted := v_inserted + 1;
    EXCEPTION
      WHEN unique_violation THEN
        v_skipped := v_skipped + 1;
        v_errors  := v_errors || jsonb_build_object(
          'row', v_i, 'reason', 'duplicate roll_number'
        );
      WHEN OTHERS THEN
        -- Rollback this student only — others continue
        v_errors := v_errors || jsonb_build_object(
          'row', v_i, 'reason', SQLERRM
        );
    END;
  END LOOP;

  RETURN jsonb_build_object(
    'inserted', v_inserted,
    'skipped',  v_skipped,
    'errors',   v_errors
  );
END;
$$;
```

### `validate_and_migrate_offline_submission`

```sql
CREATE OR REPLACE FUNCTION validate_and_migrate_offline_submission(
  p_staging_id      UUID,
  p_hmac_timestamp  TEXT,
  p_client_ts       BIGINT
)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_row             offline_submissions_staging%ROWTYPE;
  v_expected_hmac   TEXT;
  v_grace_period    INTERVAL := INTERVAL '24 hours';
BEGIN
  SELECT * INTO v_row
  FROM offline_submissions_staging
  WHERE id = p_staging_id AND migration_status = 'pending';

  IF NOT FOUND THEN
    RETURN jsonb_build_object('error', 'STAGING_NOT_FOUND');
  END IF;

  -- 1. Validate HMAC
  v_expected_hmac := encode(
    hmac(
      v_row.session_id::TEXT || ':' || p_client_ts::TEXT,
      current_setting('app.offline_sync_secret'),
      'sha256'
    ), 'hex'
  );

  IF v_expected_hmac <> p_hmac_timestamp THEN
    UPDATE offline_submissions_staging
    SET migration_status = 'rejected' WHERE id = p_staging_id;
    RETURN jsonb_build_object('error', 'HMAC_INVALID');
  END IF;

  -- 2. Validate timestamp freshness (replay prevention)
  IF to_timestamp(p_client_ts / 1000.0) < NOW() - v_grace_period THEN
    UPDATE offline_submissions_staging
    SET migration_status = 'rejected' WHERE id = p_staging_id;
    RETURN jsonb_build_object('error', 'TIMESTAMP_EXPIRED');
  END IF;

  -- 3. Migrate to student_answers (idempotent)
  INSERT INTO student_answers (
    submission_id, question_id, idempotency_key,
    selected_option, answered_at
  )
  SELECT
    s.id, v_row.question_id, v_row.idempotency_key,
    v_row.selected_option, v_row.answered_at
  FROM submissions s
  WHERE s.session_id = v_row.session_id
  ON CONFLICT (idempotency_key) DO NOTHING;

  -- 4. Mark migrated
  UPDATE offline_submissions_staging
  SET migration_status = 'migrated', migrated_at = NOW()
  WHERE id = p_staging_id;

  RETURN jsonb_build_object('ok', true);
END;
$$;
```

### `calculate_results`

```sql
CREATE OR REPLACE FUNCTION calculate_results(p_paper_id UUID)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_session         RECORD;
  v_total_q         INTEGER;
  v_correct         INTEGER;
  v_score           NUMERIC(5,2);
  v_grade           TEXT;
  v_label           TEXT;
  v_recalculated    INTEGER := 0;
BEGIN
  SELECT COUNT(*) INTO v_total_q FROM questions WHERE paper_id = p_paper_id;

  FOR v_session IN
    SELECT id, student_id FROM submissions
    WHERE paper_id = p_paper_id AND completed_at IS NOT NULL
  LOOP
    -- Count correct answers (is_correct set from answers joined with questions)
    SELECT COUNT(*) INTO v_correct
    FROM student_answers sa
    JOIN questions q ON q.id = sa.question_id
    WHERE sa.submission_id  = v_session.id
      AND sa.selected_option = q.correct_option;

    v_score := ROUND((v_correct::NUMERIC / NULLIF(v_total_q, 0)) * 100, 2);

    -- Lookup grade from grade_boundaries (never hardcoded)
    SELECT grade, label INTO v_grade, v_label
    FROM grade_boundaries
    WHERE institution_id = (
      SELECT institution_id FROM exam_papers WHERE id = p_paper_id
    )
    AND assessment_type IN (
      (SELECT type FROM exam_papers WHERE id = p_paper_id), 'ALL'
    )
    AND v_score BETWEEN min_score AND max_score
    ORDER BY assessment_type DESC
    LIMIT 1;

    -- Update is_correct on each answer
    UPDATE student_answers sa
    SET is_correct = (sa.selected_option = q.correct_option)
    FROM questions q
    WHERE sa.question_id = q.id
      AND sa.submission_id = v_session.id;

    -- Update submission score and grade
    UPDATE submissions
    SET score = v_score, grade = v_grade, result_published_at = NULL
    WHERE id = v_session.id;

    v_recalculated := v_recalculated + 1;
  END LOOP;

  RETURN jsonb_build_object('recalculated', v_recalculated);
END;
$$;
```

### `get_live_monitor_data`

```sql
CREATE OR REPLACE FUNCTION get_live_monitor_data(p_paper_id UUID)
RETURNS TABLE (
  student_id        UUID,
  full_name         TEXT,
  status            TEXT,
  answers_submitted INTEGER,
  total_questions   INTEGER,
  last_seen_at      TIMESTAMPTZ,
  completed_at      TIMESTAMPTZ,
  sync_status       TEXT
)
LANGUAGE sql
SECURITY DEFINER
STABLE  -- result may be cached within a single transaction
AS $$
  SELECT
    st.id                            AS student_id,
    st.full_name,
    CASE
      WHEN sub.completed_at IS NOT NULL THEN 'submitted'
      WHEN sub.id IS NULL               THEN 'waiting'
      ELSE 'in_progress'
    END                              AS status,
    COUNT(sa.id)::INTEGER            AS answers_submitted,
    (SELECT COUNT(*) FROM questions WHERE paper_id = p_paper_id)::INTEGER
                                     AS total_questions,
    sub.updated_at                   AS last_seen_at,
    sub.completed_at,
    sub.sync_status
  FROM students st
  LEFT JOIN submissions sub
    ON sub.student_id = st.id AND sub.paper_id = p_paper_id
  LEFT JOIN student_answers sa
    ON sa.submission_id = sub.id
  WHERE st.level_id = (
    SELECT level_id FROM exam_papers WHERE id = p_paper_id
  )
  AND st.deleted_at IS NULL
  GROUP BY st.id, st.full_name, sub.id, sub.completed_at,
           sub.updated_at, sub.sync_status
  ORDER BY st.full_name;
$$;
-- Optimised for 2500+ students: single JOIN, GROUP BY on indexed columns
-- Called by admin monitor every 5s via Supabase RPC (not real-time query per row)
```
