## 1. Colour Contrast Ratios

Minimum target: **WCAG 2.2 AAA** (7:1 normal text · 4.5:1 large text).  
All values below must be verified with a calibrated contrast analyser before deployment.

### Passing Combinations

| Foreground | Background | Ratio | Level | Context |
|-----------|-----------|-------|-------|---------|

| Foreground | Background | Ratio | Reason |
|-----------|-----------|-------|--------|
| `#FF6B6B` | `#FFFFFF` | 3.6:1 | ❌ **FAILS AA** — prohibited in all contexts |

## 2. ARIA Patterns — Component Reference

### 2.1 Exam Timer

```tsx
// src/components/assessment-engine/exam-timer.tsx
<div
  role="timer"
  aria-label="Time remaining"
  aria-live="polite"
  aria-atomic="true"
>
  {/* DM Mono formatted mm:ss */}
  {formatTime(secondsRemaining)}
</div>
```

**Announcement strategy:**
- Screen reader announces timer value **only** at 5-minute and 1-minute thresholds
- NEVER announce on every second — constant updates saturate the audio channel and destroy exam concentration
- Implementation: use a ref to track last-announced threshold; fire manual `aria-live` injection only at threshold crossings

```tsx
// Announce ONLY at thresholds — never continuously
useEffect(() => {
  if (secondsRemaining === 300) announceToSR('5 minutes remaining');
  if (secondsRemaining === 60)  announceToSR('1 minute remaining');
}, [secondsRemaining]);
```

### 2.2 Network Banner

```tsx
// src/components/assessment-engine/network-banner.tsx
<div
  role="alert"
  aria-live="assertive"
  aria-atomic="true"
  aria-label="Network status"
>
  The network is taking a quick break. Test paused. Teacher knows.
</div>
```

> `aria-live="assertive"` interrupts the screen reader's current utterance immediately — mandatory because the student must know the exam has paused. This is one of the few contexts where assertive is justified.

### 2.3 Skeleton Screens

```tsx
// Applied to every page/section while data is loading
<section
  aria-busy="true"
  aria-label="Loading content"
>
  <div className="skeleton" aria-hidden="true" />
  <div className="skeleton" aria-hidden="true" />
</section>

// Switch to false when data resolves — React re-render handles this
<section
  aria-busy="false"
  aria-label="Student directory"
>
  {/* real content */}
</section>
```

- Individual skeleton blocks: `aria-hidden="true"` (meaningless to screen reader)
- Container element: `aria-busy="true"` (announces "busy" to AT; clears on data load)
- No `aria-label` change needed — same element, attribute flips

### 2.4 MCQ Answer Grid

```tsx
// src/components/assessment-engine/mcq-grid.tsx
<fieldset>
  <legend className="sr-only">Answer choices</legend>
  <div
    role="radiogroup"
    aria-label="Answer choices"
    aria-required="true"
  >
    {options.map((option, i) => (
      <button
        key={option.id}
        role="radio"
        aria-checked={selectedOption === option.id}
        aria-label={`Option ${LABELS[i]}: ${option.value}`}
        onClick={() => handleSelect(option.id)}
        className="mcq-option"
      >
        <span aria-hidden="true">{LABELS[i]}</span>
        <span className="mcq-value">{option.value}</span>
      </button>
    ))}
  </div>
</fieldset>
```

- `role="radiogroup"` groups logically for AT navigation
- `aria-checked` reflects selection state (not `aria-selected` — that's for listboxes)
- `aria-label="Option A: 552"` gives full context on focus — AT reads "Option A: five hundred fifty-two, radio button, 1 of 4"
- Option letters (`A`, `B`, `C`, `D`) in `aria-hidden` span — the full aria-label already includes them

### 2.5 Live Monitor Table

```tsx
// src/components/monitor/live-monitor-dashboard.tsx
<table
  role="table"
  aria-label="Student exam status — live"
  aria-describedby="monitor-caption"
>
  <caption id="monitor-caption" className="sr-only">
    Updates automatically as students submit and connect.
  </caption>
  <thead>
    <tr>
      <th scope="col" aria-sort="ascending">Name</th>
      <th scope="col" aria-sort="none">Status</th>
      <th scope="col" aria-sort="descending">Last Seen</th>
    </tr>
  </thead>
  <tbody aria-live="polite" aria-relevant="additions text">
    {/* rows render here — AT receives polite updates on row changes */}
  </tbody>
</table>
```

- `aria-sort` on sortable `<th>` — updates to `"ascending"` / `"descending"` / `"none"` on click
- `aria-live="polite"` on `<tbody>` — AT announces row additions without interrupting
- `aria-relevant="additions text"` — only new rows and text changes announced; removals silent

### 2.6 Sidebar Navigation

```tsx
// src/components/layout/admin-sidebar.tsx
<nav
  role="navigation"
  aria-label="Main navigation"
>
  <ul role="list">
    {navItems.map(item => (
      <li key={item.href}>
        <a
          href={item.href}
          aria-current={isActive(item.href) ? 'page' : undefined}
        >
          <item.Icon aria-hidden="true" size={18} />
          <span>{item.label}</span>
          {item.isLive && (
            <span
              className="live-badge"
              aria-label="Live exam in progress"
            >
              LIVE
            </span>
          )}
        </a>
      </li>
    ))}
  </ul>
</nav>
```

- `aria-current="page"` — standard pattern; AT announces "current page" on active item
- `role="navigation"` with unique `aria-label` — prevents conflict with any secondary nav
- Lucide icons: `aria-hidden="true"` (decorative) — the text label carries all meaning
- LIVE badge: explicit `aria-label` describes what the badge means (not just "LIVE")

### 2.7 Modals and Sheets (shadcn/ui)

shadcn/ui `<Dialog>` and `<Sheet>` wrap Radix UI primitives that handle most ARIA automatically. Mandatory verification checklist for every modal:

```tsx
// Dialog automatically provides:
// role="dialog" | aria-modal="true" | aria-labelledby | aria-describedby

<Dialog>
  <DialogContent
    aria-labelledby="dialog-title"
    aria-describedby="dialog-description"
  >
    <DialogTitle id="dialog-title">Delete Level</DialogTitle>
    <DialogDescription id="dialog-description">
      This will soft-delete the level. Historical data is preserved.
    </DialogDescription>
    {/* content */}
    <DialogClose asChild>
      <Button aria-label="Close dialog">Cancel</Button>
    </DialogClose>
  </DialogContent>
</Dialog>
```

**Focus trap — mandatory:**
- On open: focus moves to first focusable element inside dialog (Radix handles automatically)
- Tab/Shift+Tab: cycles within dialog bounds — no escape to background content
- On close (Esc or Close button): focus returns to the element that triggered the dialog
- Verify with keyboard-only testing: `Tab`, `Shift+Tab`, `Escape` must all work correctly

```tsx
// Explicitly restore focus on close if Radix doesn't handle edge case
const triggerRef = useRef<HTMLButtonElement>(null);

<Dialog onOpenChange={(open) => {
  if (!open) {
    // Radix restores focus automatically, but ensure ref is set for edge cases
    setTimeout(() => triggerRef.current?.focus(), 0);
  }
}}>
```

### 2.9 Form Inputs

```tsx
// All form inputs must have an associated <label>
<div>
  <label htmlFor="student-name" className="form-label">
    Full Name <span aria-label="required">*</span>
  </label>
  <input
    id="student-name"
    type="text"
    aria-required="true"
    aria-invalid={errors.name ? 'true' : 'false'}
    aria-describedby={errors.name ? 'name-error' : undefined}
```

## 3. Reduced Motion

All non-essential animations must be gated behind `prefers-reduced-motion`.

### Global Rule

```css
/* globals.css — wraps ALL decorative animations */
@media (prefers-reduced-motion: no-preference) {
  .breathing-circle {
    animation: breathe 4s ease-in-out infinite;
  }

  .pulse-ring {
    animation: pulse-ring 2s infinite;
  }

  .skeleton {
    animation: shimmer 1.5s ease-in-out infinite;
  }

  .nav-transition {
    transition: background-color 150ms ease, color 150ms ease;
  }

  .card-mount {
    animation: slide-up 400ms cubic-bezier(.25,1,.5,1);
  }

  .confetti-canvas {
    display: block;
  }
}

/* When reduced motion is preferred — all decorative anims off */
@media (prefers-reduced-motion: reduce) {
  .confetti-canvas {
    display: none; /* suppress confetti entirely */
  }
  /* All other animation classes: browser default strips animation/transition */
}
```

### Exceptions — always instant regardless of preference

| Element | Reason |
|---------|--------|
| Timer digit updates | Data — not decoration. Delay = misinformation. |
| Flash Anzan number swap | `transition: none !important` — architectural requirement, not animation |
| MCQ selection state change | Instant feedback required for accurate tapping |
| Network banner appearance | Safety-critical information — must appear immediately |

### React utility hook

```tsx
// src/hooks/use-reduced-motion.ts
export function useReducedMotion(): boolean {
  const [prefersReduced, setPrefersReduced] = useState(
    () => window.matchMedia('(prefers-reduced-motion: reduce)').matches
  );

  useEffect(() => {
    const mql = window.matchMedia('(prefers-reduced-motion: reduce)');
    const handler = (e: MediaQueryListEvent) => setPrefersReduced(e.matches);
    mql.addEventListener('change', handler);
    return () => mql.removeEventListener('change', handler);
  }, []);

  return prefersReduced;
}

// Usage in Submission Confirmation — suppress confetti if preferred
function SubmissionCard() {
  const reducedMotion = useReducedMotion();
  return (
    <>
      {!reducedMotion && <ConfettiCanvas />}
      <Card>...</Card>
    </>
  );
}
```

## 4. Paediatric Motor Requirements

Children ages 6–18 have significantly wider variance in fine motor precision than adults. All interactive targets and timing delays are engineered around this constraint.

### Touch Target Minimums

| Component | Minimum Size | Standard | Notes |
|-----------|-------------|----------|-------|
| MCQ option buttons | **64px × 64px** | Apple HIG 44px | Exceeds HIG — mandatory Fitts Law floor |
| Navigation items (sidebar) | 40px height, full-width | Apple HIG 44px | Full-width maximises Fitts target area |
| Toolbar / FAB buttons | 48px | Apple HIG 44px | CTAs enlarged for primary actions |
| Filter chips | 32px height | — | Secondary — not primary motor path |
| Icon-only buttons | 44px × 44px | Apple HIG 44px | Padding added to icon hit zone |
| Begin Flash button | 56px height, full-width | — | Deliberately oversized — single point of focus |

> **Never** shrink MCQ options below 64px for any viewport size. On small screens, reduce grid columns before reducing cell size.

### Fitts Law Verification

```
Fitts Law Index of Difficulty: ID = log₂(2D / W)
  D = distance from current touch point to target centre
  W = target width
  
For MCQ grid (200px max distance, 64px button width):
  ID = log₂(2 × 200 / 64) = log₂(6.25) ≈ 2.64 bits

Target: ID ≤ 2.5 bits → maintained by keeping MCQ within 200px of equation.
If equation overflows available height → shrink equation font before increasing distance.
```

### 1200ms Cooldown (Inter-tap Interval Protection)

Children under stress exhibit inter-tap intervals of ~500ms. Without a cooldown, a child who taps an MCQ option will often accidentally confirm a wrong answer on the "Next →" confirmation button that appears at the same position.

```tsx
// src/hooks/use-input-cooldown.ts
// Uses RAF + performance.now() delta — NOT setTimeout.
// 09_fsd.md §2 QUESTION_COOLDOWN: "performance.now() delta, not setTimeout"
// setTimeout fires late on throttled main threads (GC, low-end devices) —
// a child's tap during the extra delay window is silently discarded.
export function useInputCooldown(durationMs = 1200) {
  const [isCooling, setIsCooling] = useState(false);
  const rafHandleRef = useRef<number | null>(null);
  const startTimeRef  = useRef<number>(0);

  const triggerCooldown = useCallback(() => {
    setIsCooling(true);
    startTimeRef.current = performance.now();

    // Cancel any previous cooldown still running
    if (rafHandleRef.current !== null) {
      cancelAnimationFrame(rafHandleRef.current);
    }

    function tick(): void {
      const elapsed = performance.now() - startTimeRef.current;
      if (elapsed >= durationMs) {
        setIsCooling(false);
        rafHandleRef.current = null;
      } else {
        rafHandleRef.current = requestAnimationFrame(tick);
      }
    }

    rafHandleRef.current = requestAnimationFrame(tick);
  }, [durationMs]);

  // Cleanup on unmount
  useEffect(() => {
    return () => {
      if (rafHandleRef.current !== null) {
        cancelAnimationFrame(rafHandleRef.current);
      }
    };
  }, []);

  return { isCooling, triggerCooldown };
}

// Apply in assessment engine — invisible overlay, not visual feedback
{isCooling && (
  <div
    aria-hidden="true"
    style={{
      position: 'fixed', inset: 0,
      pointerEvents: 'none', // blocks touch/click passthrough
      zIndex: 9998,
    }}
  />
)}
```

- The overlay is invisible — no visual distraction
- `aria-hidden="true"` — screen readers unaffected
- Applies to the entire assessment canvas during cooldown, not just the MCQ grid

### No Hover-Only Interactions

Tablet users (the primary hardware platform for this demographic) cannot hover. All interactions that trigger on hover must have a touch/focus equivalent.

```tsx

</TooltipProvider>
```

## 5. Screen Reader Announcement Strategy

### sr-announcer.tsx — Dedicated Announcement Component

All exam-context screen reader messages must be routed through a single, controlled component. This prevents multiple `aria-live` regions competing and producing overlapping announcements.

```tsx
// src/components/a11y/sr-announcer.tsx
import React, { useState, useImperativeHandle, forwardRef } from 'react';

export interface SRAnnouncerHandle {
  announce: (msg: string) => void;
}

// MUST use forwardRef — useImperativeHandle requires a ref parameter.
// Without forwardRef, ref is undefined and useImperativeHandle is a no-op.
// All exam-context AT announcements depend on this ref being callable.
export const SRAnnouncerRef = forwardRef<SRAnnouncerHandle, {}>((_, ref) => {
  const [message, setMessage] = useState<string>('');

  // Expose via ref for imperative calls from exam engine
  useImperativeHandle(ref, () => ({
    announce(msg: string) {
      setMessage(''); // clear first — forces re-announcement if same string
      requestAnimationFrame(() => setMessage(sanitizeForSR(msg)));
    }
  }));

  return (
    <div
      aria-live="polite"
      aria-atomic="true"
      aria-relevant="additions"
      className="sr-only"
    >
      {message}
    </div>
  );
});

SRAnnouncerRef.displayName = 'SRAnnouncerRef';
```

**Sanitization — mandatory:**
```tsx
// src/components/a11y/sr-announcer.tsx
function sanitizeForSR(raw: string): string {
  // 1. Strip all HTML tags
  // 2. Decode HTML entities
  // 3. Never pass raw DB strings, error codes, or UUIDs
  return raw
    .replace(/<[^>]*>/g, '')
    .replace(/&amp;/g, '&')
    .replace(/&lt;/g, '<')
    .replace(/&gt;/g, '>');
}
```

### What Gets Announced

| Event | Announcement text | Region |
|-------|------------------|--------|
| Question change | "Question 3 of 20" | `aria-live="polite"` |
| 5 min warning | "5 minutes remaining" | `aria-live="polite"` |
| 1 min warning | "1 minute remaining" | `aria-live="polite"` |
| Network offline | "Your network paused. Your work is saved. Your teacher knows." | `role="alert"` (assertive) |
| Exam paused | "Assessment paused — focus lost" | `role="alert"` (assertive) |
| Submitted | "Assessment submitted. Your answers are saved." | `aria-live="polite"` |

### What Must NEVER Be Announced

| Prohibited | Reason |
|------------|--------|
| Raw error codes (`Error 503`) | Meaningless and alarming to students |
| Timer every second | Saturates the audio channel during active concentration |
| Database IDs or UUIDs | Never expose internal data to audio stream |
| Other students' names or status | Privacy — GDPR relevant |
| Flash Anzan numbers during Phase 2 | `aria-live="off"` — Ticker Mode is the accessible alternative |

## 6. Flash Anzan Accessibility

Flash Anzan Phase 2 presents a fundamental accessibility challenge: centred rapid-flash numbers cannot be meaningfully accessed via a screen reader. The solution is a **Ticker Mode** alternative track.

### 6.1 aria-live During Phase 2

```tsx
// Phase 2 wrapper — silence screen reader during flash
<div
  role="region"
  aria-label="Flash number sequence"
  aria-live="off"    // screen reader will not announce flashing numbers
  aria-busy="true"   // conveys "something is happening" without reading it out
>
  <span className="flash-number">{currentNumber}</span>
</div>
```

> `aria-live="off"` means the region content updates will NOT be announced. The screen reader user instead uses Ticker Mode.

### 6.2 Ticker Mode — Accessible Alternative

**Triggered by:** `accessibility_flags.ticker_mode = true` on the student's profile (set by admin/teacher on the student settings tab).

```tsx
// src/components/a11y/a11y-ticker-mode.tsx

interface TickerModeProps {
  sequence: number[];
  intervalMs: number;
}

export function TickerMode({ sequence, intervalMs }: TickerModeProps) {
  // Numbers scroll horizontally at user-controlled speed
  // Speed derived from assessment delay_ms but min 800ms for readability
  const displayInterval = Math.max(intervalMs, 800);

  return (
    <div
      role="region"
      aria-label="Flash number sequence — accessible ticker"
      aria-live="polite"
    >
      {/* Horizontal scroll container */}
      <div className="ticker-track" aria-hidden="true">
        {sequence.map((n, i) => (
          <span
            key={i}
            className="ticker-item"
            style={{ animationDelay: `${i * displayInterval}ms` }}
          >
            {n}
          </span>
        ))}
      </div>

      {/* Screen reader: announces each number as it becomes current */}
      <span className="sr-only" aria-live="polite">
        {currentTickerNumber}
      </span>
    </div>
  );
}
```

**Ticker Mode properties:**
```
Number display: horizontal scroll (RTL to LTR — left-to-right reading direction)
Min display time: max(delay_ms, 800ms) — never faster than 800ms in ticker mode
Font: DM Mono 48px, #0F172A on #FFFFFF
Negative numbers: #991B1B
No transitions on number swap — instant, same as Phase 2
```

### 6.3 Low Vision — Font Size Scaling

The Flash number uses `clamp(96px, 30vh, 180px)`. This means:
- On a standard 768px viewport: 30vh = 230px → capped at 180px
- Font is always at least 30% of viewport height
- System font scaling (browser zoom / OS text size) still applies above the clamp value

## 7. Focus Management

### 7.1 Global Focus Style

```css
/* globals.css — never remove, never override with outline:none without replacement */
:focus-visible {
  outline: 3px solid #1A3829;
  outline-offset: 2px;
  border-radius: 4px;
}

/* Remove outline for mouse users only (not keyboard users) */
:focus:not(:focus-visible) {
  outline: none;
}
```

> `#1A3829` (green-800) at 3px provides ~12:1 contrast on all approved backgrounds. Meets WCAG 2.2 Success Criterion 2.4.11 (Focus Appearance — AA) and exceeds it to AAA.

### 7.2 Modal Focus Trap

Every `<Dialog>` and `<Sheet>` must trap focus within the overlay while open. Radix UI (used by shadcn/ui) handles this automatically. **Verify these 3 conditions after every modal implementation:**

1. Tab from last focusable element → wraps to first focusable element inside modal
2. Shift+Tab from first focusable element → wraps to last focusable element
3. Escape key → closes modal AND returns focus to the trigger button

### 7.3 Route Change Focus

On Next.js App Router navigation (SPA-style route changes), focus does not automatically move. Use a skip-link + focus management pattern:

```tsx
// src/app/layout.tsx — global skip link
<a
  href="#main-content"
  className="skip-link sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4"
>
  Skip to main content
</a>

// Each page — main landmark
<main id="main-content" tabIndex={-1}>
  {/* page content */}
</main>
```

On route change, programmatically shift focus to `#main-content`:
```tsx
// src/hooks/use-route-focus.ts
const pathname = usePathname();
useEffect(() => {
  document.getElementById('main-content')?.focus();
}, [pathname]);
```

### 7.4 Question Transition Focus

When the exam engine advances to the next question, focus must move to the new question's equation container — not remain on the MCQ button just clicked.

```tsx
const questionRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  // On question index change, move focus to equation container
  questionRef.current?.focus();
}, [currentQuestionIndex]);

<div
  ref={questionRef}
  tabIndex={-1}
  aria-label={`Question ${currentIndex + 1} of ${total}`}
  className="equation-container"
>
  {/* equation */}
</div>
```

## 8. Keyboard Navigation

### 8.1 Global Keyboard Shortcuts

| Shortcut | Action | Context |
|----------|--------|---------|
| `Cmd+K` / `Ctrl+K` | Open command palette | Admin panel |
| `Escape` | Close modal / sheet / palette | Any overlay |
| `Tab` / `Shift+Tab` | Navigate interactive elements | Global |
| `Enter` / `Space` | Activate button or link | Global |
| `Arrow keys` | Navigate radio group (MCQ) | Exam canvas |

### 8.2 MCQ Arrow Key Navigation

```tsx
// MCQ radio group — arrow key navigation between options
<div
  role="radiogroup"
  onKeyDown={(e) => {
    const options = ['A', 'B', 'C', 'D'];
    const currentIdx = options.indexOf(selectedOption ?? 'A');

    if (e.key === 'ArrowRight' || e.key === 'ArrowDown') {
      e.preventDefault();
      const next = (currentIdx + 1) % 4;
      setSelectedOption(options[next]);
      optionRefs[next].current?.focus();
    }
    if (e.key === 'ArrowLeft' || e.key === 'ArrowUp') {
      e.preventDefault();
      const prev = (currentIdx + 3) % 4;
      setSelectedOption(options[prev]);
      optionRefs[prev].current?.focus();
    }
  }}
>
```

### 8.3 Data Table Keyboard Navigation

```
Tab         → move to next interactive element (next row action button)
Shift+Tab   → previous interactive element
Click/Enter → activate row action
Sortable th → Enter/Space to sort; aria-sort toggles ascending/descending/none
```
